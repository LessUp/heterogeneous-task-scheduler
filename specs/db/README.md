# Database Schemas

This directory is reserved for database schema definitions and data models.

---

## Purpose

Database schemas define the structure and relationships of persistent data:

- **DBML** — Database Markup Language for relational schemas
- **SQL Migrations** — Version-controlled schema changes
- **ER Diagrams** — Entity-relationship visualizations
- **Data Models** — Logical data structure definitions

---

## Current Status

HTS is a C++ task scheduling library that does not use a database. All state is managed in memory during execution.

This directory is reserved for future extensions:

- Task execution history persistence
- Configuration storage
- Distributed state management
- Analytics and metrics storage
